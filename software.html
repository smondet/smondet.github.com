Software

I strongly believe that trusting the human brain is stupid and irresponsible … Hence, the fact that most programmers believe they can handle C++ or Python is the main reason why computers and software are such a pathetic disaster, especially regarding safety or security.

Instead, my goal is to use formal methods to make software suck as less as possible. I program everything in OCaml; by very far the best practical implementation of an advanced, strong, and static type system, with a cool ecosystem built by people who care about the code. I also keep a close eye on what will be the next step in real-world software engineering (Coq?).

I have some open-source code living around there, see Github, Bitbucket, and Gitlab.
